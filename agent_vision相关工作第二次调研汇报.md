# Agent Vision：正交视角下的细粒度像素定位调研报告

**任务背景：** 在使用 Gemini-3-Flash 完成正交照片中「桌角识别」的初步 Demo 后，针对 Agent 在物理空间感知中的精度瓶颈，对相关数据集、评价指标及 Baseline 方法进行了系统调研。

## 1. 任务定义与核心挑战

我们的目标是实现 **像素级（Pixel-level）** 的桌角识别，这在 Agent Vision 领域属于 **「Affordance Keypoint Grounding」（示能关键点定位）** 的子范畴。

### 核心挑战

- **坐标漂移：** 通用 VLM（如 Gemini/GPT）输出的归一化坐标存在量化误差。
- **空间感知：** Agent 需要从二维正交图推断三维物理交互点。
- **高精度需求：** 机器人抓取或对齐任务对误差的容忍度通常在 5–10 像素以内。

---

## 2. 核心论文与 Baseline 方法

目前学术界的主流思路已从「直接回归坐标」转向「显式位置映射」和「视觉辅助指向」。

### 关键论文推介

| 论文名称 | 核心思路 | 解决的痛点 |
| :--- | :--- | :--- |
| **Improving GUI Grounding with Explicit Position-to-Coordinate Mapping** (2025) | 提出 **RULER** 机制，在输入端引入旋转位置编码与显式刻度映射。 | 解决了模型对坐标系理解模糊、定位精度低的问题。 | 链接：https://arxiv.org/abs/2510.03230
*核心贡献*：提出了 RULER（Rotary position-to-pixeL mappER）机制。传统的 VLM（如 Gemini Flash）是直接把坐标当成文本输出，这是一种“隐式映射”。这篇文章通过在输入端加入显示的“标尺”特征，让模型像看地图刻度一样去定位，极大地提升了像素级精度。
*对我们的启示*：在识别桌角时，如果模型直出坐标不准，可以参考其在图片上叠加“虚拟刻度”或“参考锚点”的做法。
| **MOKA: Marking Open-world Keypoints for Affordance Learning** | 采用 **Visual Prompting**，先生成候选点再让模型精选。 | 降低了模型直接输出坐标的难度，利用了 VLM 的辨别能力。 | 核心思路：这篇论文提出了“标注（Marking）”的概念。它不是让模型空想坐标，而是先生成候选点，再让模型选出最准确的“示能点（Affordance Point）”。这非常适合你的“桌角识别”，因为桌角就是一个典型的示能点（抓取或对齐的参考点）。
| **MolmoPoint: Better Pointing Architecture for VLMs** (2026) | 专门为「指向（Pointing）」设计的轻量化架构，采用 Coarse-to-fine 策略。 | 在不增加计算量的前提下提升了微观特征的捕捉能力。 |核心思路：这是 Allen AI 团队最新的工作。他们认为模型不应该学习坐标系，而应该学习“指向（Pointing）”。通过一种“粗到细（Coarse-to-fine）”的注意力机制，模型可以像手指点物体一样精确定位。
| **Towards Pixel-Level VLM Perception via Simple Points Prediction (SimpleSeg)** | 简单的 **点预测（Point Prediction）** 经 **强化学习（RL）** 微调，可达到优于复杂分割模型的精度。 | 表明像素级 VLM 感知未必依赖重型分割头，「少一点、准一点」亦具竞争力。 |核心思路：证明了简单的“点预测（Point Prediction）”通过强化学习（RL）微调，可以达到比复杂的分割模型更好的精度。
**链接：** [SimpleSeg（OpenReview）](https://openreview.net/forum?id=CcQFKeUK5K)

### 方法论演进趋势

1. **Iterative Zooming（迭代缩放）：** 先定位粗略区域，自动 Crop 后二次放大识别（模拟人类凑近看的过程）。
2. **Grid-based Prompting（网格引导）：** 在原图上覆盖一层半透明的坐标网格，强制模型通过「读表」而非「感知」输出坐标。


### 精读：《Improving GUI Grounding with Explicit Position-to-Coordinate Mapping》
以下是对该论文（中文可译为「通过显式位置–坐标映射提升 GUI 定位能力」）核心内容的系统梳理，与上表中的 **RULER** 条目一一对应。

#### （1）核心问题：隐式映射的局限性

目前的视觉定位模型（如 GUI Agent）通常将 **坐标生成** 视为纯文本生成任务（例如输出 `"x=523, y=217"`）。这种方法存在两个核心问题：

- **不稳定的隐式学习：** 模型必须在没有任何显式空间引导的情况下，尝试将高维视觉特征映射到精确的数字 Token 上。
- **分辨率泛化性差：** 在特定分辨率下训练的模型，一旦部署到未见的高分辨率屏幕上，由于隐式映射函数难以外推，定位精度会大幅下降。

#### （2）核心创新：RULER 与 I-MROPE

论文提出两个互补方案，旨在将「不稳定的回归问题」转化为「稳健的参考与微调机制」。

**RULER Tokens（显式坐标标尺）**

- **定义：** 在输入序列中加入一组辅助的坐标 Token（RULER tokens）。
- **原理：** 这些 Token 类似地图上的网格线，与对应的图像块（Patches）共享位置嵌入（Positional Embeddings）。
- **机制：** 模型不再从零猜测坐标，而是先匹配位置最接近的 RULER Token，获取其参考值，再进行小范围的算术修正，从而降低高分辨率图像上的泛化难度。

**I-MROPE（交错式多维旋转位置嵌入）**

- **改进对象：** 针对标准 MROPE（常用于 Qwen2-VL 等）中存在的频率分布不均问题。
- **改进点：** 标准做法将高频与低频分别固定分配到不同维度（如仅高度或仅宽度），易导致各轴建模能力失衡。
- **方案：** I-MROPE 通过 **交错分配** 频率索引，使宽、高维度都能获得完整频谱（兼顾高频细粒度定位与低频长程依赖）。

#### （3）实验设置与数据集

- **训练数据：** **UGround** 数据集，约 77.5 万张截图、约 800 万个元素标注。
- **评估基准：**
  - **ScreenSpot / ScreenSpot-V2：** 移动、桌面与 Web 等标准 GUI 定位任务。
  - **ScreenSpot-Pro：** 23 个专业级桌面应用，更高分辨率与更大域偏移（Domain Shift），用于检验泛化。
- **实现框架：** 基于 LLaVA-NeXT（从头训练）与 Qwen2.5-VL 7B（微调）等。

#### （4）主要实验结果

- **精度：** 在最具挑战性的 **ScreenSpot-Pro** 上，仅通过微调，准确率由 **31.1% 提升至 37.2%**。
- **泛化：** RULER 的「参考–复制」机制在处理 **超过训练分辨率** 的图像时表现有效。
- **开销：** RULER Token 计算开销很小；即便 **8K** 截图，RULER 也仅占视觉 Token 总数的约 **0.2%**。

#### （5）结论与对本任务（正交桌角识别）的启示

论文表明：**显式空间引导（如 RULER）比依赖预训练「涌现」的隐式空间感知更可靠**。

对本 Demo 的直接启发：

1. **方法：** 可在正交图上显式加入类似 RULER 的参考坐标系（或与「Visual Anchor / 网格」结合），避免让 Gemini **盲回归** 像素坐标。
2. **验证：** 其采用的 **Element Accuracy**（预测点是否落在真值框内）及 **不同分辨率下的稳健性** 测试，可借鉴为桌角定位后的评价与消融设计。

*可进一步细化：RULER 类 Token 在 API 调用中的 Prompt 构造方式，以及在本任务中如何实现「标尺」式输入（图像叠加刻度 / 锚点说明等）。*


### 精读：MOKA（Marking Open-world Keypoints for Affordance Learning）

**MOKA（Marking Open-world Keypoint Affordances）** 利用预训练视觉语言模型（VLM，如 GPT-4V）处理机器人开放世界操纵：将复杂控制问题转化为 VLM 擅长的 **视觉问答（VQA）** 形式，与上表中 **Visual Prompting / 候选点再精选** 的条目对应。

#### （1）核心挑战与方案思路

- **挑战：** 互联网规模预训练的 VLM 具备常识，但对 **三维空间**、接触物理与机器人控制的刻画仍不足。
- **核心方案：** 建立 **点对点的示能（Affordance）表示**，作为视觉观测与机器人动作之间的桥梁。

#### （2）技术架构：分层视觉提示

MOKA 采用层级框架处理自由文本指令与视觉观测：

- **高层推理：** 将复杂自然语言指令（如「擦掉桌子上的垃圾」）分解为子任务；识别每步涉及的物体（如手中物体 \(o_{\text{in-hand}}\)、环境物体 \(o_{\text{unattached}}\)）与运动方向。
- **底层推理：** 使用 **基于标记的视觉提示（Mark-based Visual Prompting）**，在 2D 图像上标注候选点与网格；VLM 以 **选择题** 方式选出最优关键点，再经深度图投影到三维并转化为可执行轨迹。

#### （3）关键点示能表示（Affordance Representation）

MOKA 用统一的一组点描述多种操纵技能：

| 类型 | 含义 |
| :--- | :--- |
| **抓取关键点** \(x_{\text{grasp}}\) | 夹爪应握持物体的部位 |
| **功能关键点** \(x_{\text{function}}\) | 物体（或夹爪）上用于与目标接触的部位 |
| **目标关键点** \(x_{\text{target}}\) | 环境物体上被接触的部位 |
| **路径点** \(x_{\text{pre-contact}}, x_{\text{post-contact}}\) | 接触前后的运动规划；常借助图像上的 **\(5 \times 5\)** 网格（类似棋盘格记法）定位 |

#### （4）性能提升与数据蒸馏

- **上下文学习（In-context Learning）：** 将 2–3 条成功轨迹作为示例写入 Prompt，可显著提升推理成功率。
- **策略蒸馏（Policy Distillation）：** 将 MOKA 生成的成功轨迹作为演示，微调专用视觉–运动策略（如 Octo），提高执行效率。

#### （5）实验结果

在擦桌、清理手表、准备礼物、包装笔记本等桌面任务上：零样本（Zero-shot）下整体优于 Code-as-Policies、VoxPoser 等；对物体类别、初始布局与语言指令变化 **鲁棒性** 较好。失败主要分为 **推理失败**（VLM 选错点）与 **执行失败**（控制精度不足、滑脱等）。

#### （6）局限性

- VLM 对三维空间与接触物理的理解仍偏粗，限制更复杂运动的生成。
- 调用 VLM **延迟高**，不利于实时动态操纵或依赖高频闭环的场景。

#### （7）与本任务及 RULER 的关联

- **汇报可强调：** MOKA 将 **「坐标回归」** 转为 **「视觉点选 / 网格上的离散选择」**，减轻直接输出连续坐标的难度，与 RULER 论文中 **显式坐标映射** 在「提高定位可信度」上 **思路相近、路径不同**（MOKA 偏 **候选 + 判别**；RULER 偏 **标尺参考 + 微调**）。
- **简要对比：**

| 维度 | RULER（GUI Grounding） | MOKA（机器人操纵） |
| :--- | :--- | :--- |
| 定位策略 | 序列侧 RULER Token + 与 Patch 对齐的位置嵌入，参考值 + 算术修正 | 图像上标记候选点/网格，VLM 做选择题 |
| 典型场景 | 屏幕 GUI、高分辨率外推 | 桌面操纵、开放词汇指令 |
| 与本 Demo | 正交图加「标尺/锚点」、分辨率稳健性 | 桌角可先 **粗候选（分割/角点）+ VLM 精选**，或网格离散化 |

*可进一步：在桌角任务上设计「分割初定位 + Gemini 在局部网格内点选」的消融，与纯坐标回归对照。*


### 精读：SimpleSeg（*Towards Pixel-Level MLLM Perception via Simple Points Prediction*）

以下梳理主要依据该文在 **ICLR 2026** 评审阶段的 **反驳补充材料（Rebuttal Supplementary Materials）**，与上表中 **SimpleSeg** 一行对应；材料中展示的核心模型 **SimpleSeg** 通过直接预测 **简单点集（像素坐标）** 构成多边形，实现像素级视觉感知。

#### （1）核心方法论

工作质疑「唯复杂分割架构不可」的路径，表明多模态大模型（MLLM）可通过 **坐标点预测（Points Prediction）** 完成精细分割：模型输出 **归一化多边形顶点序列**，例如形如 `[[[x1, y1], [x2, y2], ...]]` 的结构化坐标，而非先训重型 Mask 头再后处理。

#### （2）补充材料中的能力展示（图示概览）

**A. 背景与遥感（Figure 1）**

- **背景语义分割：** 能勾勒「海」等背景区域。
- **负样本：** Prompt 指向图中不存在的类别（如「狗」）时，可反馈 **图中无目标对象**。
- **遥感 / 俯视分割：** 俯视场景下可定位 **细长机场跑道** 等多边形边界——与 **正交视角、细长结构** 的桌沿/桌角任务 **可参考对照**。

**B. 全景与多目标（Figure 2）**

- **全景分割：** 同时输出物体（如猫）与背景（楼梯、地板、天空等）的多边形顶点。
- **多实例：** 单图中多只目标可输出 **彼此独立** 的点集。
- **多部分：** 可区分同一物体的不同组成部分。

#### （3）局限性与失败案例（Figure 3）

- **带孔物体：** 如甜甜圈，整体与孔洞可分别识别，但 **合并为单一逻辑多边形** 时仍可能不稳。
- **纹理混淆：** 斑马等 **强重复纹理** 目标上，多边形边缘易出现模糊或偏离真实轮廓。

#### （4）对本任务（正交桌角识别）的启示

1. **「坐标点即感知」：** SimpleSeg 表明对 Agent Vision 而言，让模型 **直接预测关键点/顶点坐标** 是可行路线，相对「生成稠密 Mask 再换算」可更高效。
2. **评价可扩展：** 补充材料强调 **负样本** 与 **遥感（类正交）图像**——本 Demo 可增加：**无桌面场景下模型是否仍乱标桌角**（幻觉/假阳性）。
3. **Baseline 层级：** **直接输出顶点坐标** 可作为最简 Baseline；经 **RL 微调** 的 SimpleSeg 类方法可作为 **更强对比**（与纯 API 调用 Gemini 形成梯度）。

**链接：** [OpenReview（SimpleSeg）](https://openreview.net/forum?id=CcQFKeUK5K)

*可进一步：按补充材料中的 **JSON 多边形点集** 约定，为 Gemini-3-Flash 撰写专门用于「桌角点选」的 System Prompt，并与网格/锚点方案做对照。*

## 3. 数据集调研

面向 Agent 的数据集与面向 LLM 预训练的数据集有显著差异：

- **规模与密度：** Agent 数据集规模通常在 **1 万–20 万** 张，但单张图片的标注极度密集（精确到像素点的坐标、法向量、接触点信息）。
- **代表性数据集：**
  - **RealVLG-11B (2026)：** 包含 16.5 万张真实物理场景，重点在于接触点（Contact Points）标注。
  - **PixMo-Point：** 针对自然图像中各类几何关键点（角、边缘中心）的密集坐标标注。
  - **ScreenSpot Pro：** 虽为 GUI 任务，但其跨分辨率定位的思路（Relative vs Absolute）对我们具有高度参考价值。

---

## 4. 评价指标（Evaluation Metrics）

针对点定位任务，我们不再使用 IoU，而是采用以下指标衡量 Agent 的「手眼协同」能力：

1. **Pixel Distance Error（L2）：** 预测坐标与真值坐标之间的欧几里得距离。
2. **Accuracy @ δ（Success Rate）：** 在允许误差范围 δ（如图片长边的 1% 或 5px）内的样本占比。
3. **Resolution Robustness：** 测试在不同分辨率输入下，模型定位的一致性评分。

## 5. 后续实验计划

基于调研结果，我计划从以下两个方向推进方法层面的优化：

- **方案一（无需微调）：** 引入 **Visual Anchor Prompting**。在正交图中预埋 4 个参考锚点，让 Gemini 学习基于相对参考系输出桌角坐标，以抵消其原生的非线性偏移。
- **方案二（数据集构建）：** 收集约 500 张正交视角桌角图，采用 **JSON-L** 格式（包含 `image_id`, `point_coords`, `point_type`）建立评估集，作为我们方法的 Benchmark。
