# 桌面四角点识别进展汇报

**汇报对象**：师兄  
**汇报内容**：基于 Gemini Robotics-ER 1.5 的桌角识别当前方案、所用提示词及识别成果说明。  
**说明**：目前识别效果尚未达到师兄要求的精度标准，本报告主要汇报已采用的提示词设计与当前实现情况，便于后续在师兄指导下继续改进。

---

## 一、背景与目标

- **任务**：在 Blender 正交相机拍摄的图像中，精确定位桌面（tabletop）的四个角点，输出归一化坐标，并保证四点构成平行四边形。
- **模型**：Gemini Robotics-ER 1.5（`gemini-robotics-er-1.5-preview`），单次对话内完成检测。
- **当前状态**：已完成提示词设计、单图/参考图双模式、解析与可视化流程；识别稳定性与精度尚未达到师兄设定的标准，需要后续迭代。

---

## 二、当前运用的提示词

目前采用**两套提示词**，根据是否使用参考图在代码中二选一调用。

### 2.1 主提示词（单图模式）

在**仅传入一张待检测图**时使用，主要结构如下。

| 模块 | 内容要点 |
|------|----------|
| **场景设定** | 说明图像来自正交相机（如 Blender），桌面在图中呈平行四边形、无透视畸变。 |
| **任务定义** | 只定位「上桌面」的四个角点；排除地面、桌腿、桌上物体上的点。 |
| **桌子特征** | 平面边界、清晰桌沿、桌腿位置、桌上物体、均匀区域、阴影/高光等 6 条视觉线索，辅助模型找到桌面边界。 |
| **角点定义** | 每个角点 = **三条线段相交**：两条桌沿 + 该角处的竖直棱；只考虑上桌面上的点。 |
| **角点顺序（与图像绑定）** | 为避免四个角混淆，将角点与图像坐标明确对应：**corner_0** = 最靠近图像左上（y 最小，同则 x 最小）；**corner_1** = 右上（y 最小、x 最大）；**corner_2** = 右下（y、x 最大）；**corner_3** = 左下（y 最大、x 最小）。顺序为左上→右上→右下→左下（顺时针）。 |
| **四角区分** | 用「哪两条桌边 + 竖直」区分：corner_0 = 上边与左边交点，corner_1 = 上边与右边交点，corner_2 = 右边与下边交点，corner_3 = 下边与左边交点。 |
| **强约束** | 边端点、非物体角、边界拐点、凸四边形；禁止使用桌面中心、物体中心、桌腿下方等。 |
| **遮挡处理** | 若第四角被遮挡，用平行四边形公式计算：corner_3 = corner_0 + (corner_2 - corner_1)。 |
| **输出格式** | 仅输出 JSON 数组，4 个 `{ "point": [y, x], "label": "corner_i" }`，坐标归一化 0–1000，四点必须构成平行四边形。 |

### 2.2 参考图提示词（双图模式）

在**同时传入参考图 table.png（红点标出四角）与待检测图**时使用。

| 模块 | 内容要点 |
|------|----------|
| **双图说明** | 第一张：带红点标记的示例，定义「桌角点」的视觉含义；第二张：待检测图。 |
| **角点顺序** | 与主提示词一致：corner_0 = 左上，corner_1 = 右上，corner_2 = 右下，corner_3 = 左下，顺序与参考图中红点对应。 |
| **四角区分** | 再次强调每个角由不同一对桌边 + 竖直相交，三个方向组合不同，不可互换 label。 |
| **输出** | 只返回**第二张图**的四个角点，格式与主提示词相同，且必须成平行四边形。 |

### 2.3 模型与调用参数

- **模型**：`gemini-robotics-er-1.5-preview`  
- **temperature**：0.2  
- **thinking_budget**：2048（用于几何与顺序推理）  
- **输出规范**：坐标归一化 0–1000，每点为 `[y, x]`，图像左上 (0,0)、右下 (1000,1000)。

## 三、识别成果与实现情况

### 3.1 已实现功能

- **单图模式**：仅上传待检测图，由主提示词引导模型输出四个角点。
- **参考图模式**：可选传入 table.png 与待检测图，由参考图提示词引导，使「桌角点」定义与师兄提供的示例一致。
- **解析与几何兜底**：从模型返回中解析 JSON；若返回带 `corner_0`～`corner_3` 的 label，则按 label 排序后再做平行四边形约束；若为 3 个点则用公式补全第 4 点；若为 4 个点则用前三点重算第四点，保证输出严格为平行四边形。
- **可视化与保存**：在图上绘制四个红点并顺次连成红色四边形，保存为 `{文件名}_table_marked.png`，不覆盖原图。

### 3.2 测试图像

- 支持对 table1.png、table2.jpg、table3.jpg、table4.png、table5.jpg 及参考图 table.png 进行测试；参考图模式下使用 table.png 辅助对其它图的识别。

### 3.3 代码与提示词位置

- 主提示词：`gemini_table_corner_vision_ai.py` 中变量 `PROMPT`  
- 参考图提示词：同文件中 `PROMPT_WITH_REFERENCE`  
- 运行方式：在项目目录下执行 `streamlit run gemini_table_corner_vision_ai.py`，需预先设置环境变量 `GEMINI_API_KEY`。

## 四、不足与原因分析

- **识别精度未达师兄要求**：当前在部分图像上仍存在角点偏移、个别角点与真实桌角不对齐或顺序偶发错乱等情况，尚未达到师兄设定的识别标准。
- **可能原因**：  
  1. 模型对「三条线段相交」「上桌面边界」等抽象描述的依从性有限，不同场景下表现不稳定；  
  2. 正交/等距视角下桌面形状、遮挡、光照差异较大，单一套提示词难以覆盖所有情况；  
  3. 仅依赖提示词与思考预算，未引入额外几何约束或后处理（如边缘检测、模板匹配等），鲁棒性有限。
